# Content Management Guide

This guide explains how to add and manage content in the portfolio using Sanity CMS.

## Accessing Sanity Studio

1. Navigate to `/admin` on your site
2. You'll see the Sanity Studio interface
3. Use the sidebar to navigate between content types

## Content Types

### Projects

Projects showcase your work and case studies.

**Required Fields:**
- Title
- Slug (auto-generated from title)
- Date
- Description

**Recommended Fields:**
- Cover Image
- Tech Stack
- Role
- Impact
- Problem Statement
- Research Insights
- Process
- Metrics (before/after/improvement)
- Outcome

**Tips:**
- Use high-quality cover images (1200x800px recommended)
- Write compelling impact statements (e.g., "Reduced load time by 30%")
- Include detailed process documentation
- Add metrics to show quantifiable results

### Research

Research items document your UX/UI experiments and design process.

**Required Fields:**
- Title
- Slug
- Date

**Recommended Fields:**
- Hypothesis
- Methodology
- Outcome
- Tools
- Sketches
- Wireframes
- Final UI

**Tips:**
- Document your research process thoroughly
- Include visual progress (sketches → wireframes → final)
- Highlight key insights and learnings

### Writings

Blog posts and technical articles.

**Required Fields:**
- Title
- Slug
- Published At
- Body

**Recommended Fields:**
- Excerpt
- Cover Image
- Reading Time
- Tags

**Tips:**
- Write engaging excerpts for previews
- Use tags for categorization
- Set accurate reading times
- Include relevant cover images

### Build Logs

Quick updates and changelog entries.

**Required Fields:**
- Title
- Date

**Recommended Fields:**
- Summary

**Tips:**
- Keep logs concise
- Update regularly to show active development
- Highlight major milestones

### Testimonials

Client and collaborator testimonials.

**Required Fields:**
- Author
- Quote

**Recommended Fields:**
- Role
- Company
- Project (reference)

**Tips:**
- Use authentic testimonials
- Link to relevant projects
- Include author credentials

### Site Settings

Global site configuration.

**Fields:**
- Site Title
- Tagline
- Mission Statement
- Design Philosophy
- Logo
- GitHub Username
- Voice Intro File
- Ambient Music File
- Social Links
- SEO Settings

**Tips:**
- Keep tagline concise and memorable
- Update social links regularly
- Optimize SEO settings for search engines

## Image Guidelines

### Cover Images
- Recommended size: 1200x800px
- Format: JPG or WebP
- Optimize before uploading
- Use consistent aspect ratio

### Logo
- Recommended size: 512x512px
- Format: PNG with transparency
- Use high-resolution version

### Research Images
- Organize by stage (sketches, wireframes, final)
- Use consistent sizing
- Add captions for context

## SEO Best Practices

### Project Pages
- Write compelling descriptions
- Use relevant keywords naturally
- Include alt text for images
- Add structured data if possible

### Writing Pages
- Optimize titles for search
- Write meta descriptions
- Use relevant tags
- Include internal links

## Content Workflow

1. **Draft Content**: Create content in Sanity Studio
2. **Review**: Check preview in studio
3. **Publish**: Make content live
4. **Update**: Regularly update and refine content

## Tips for Success

- Keep content fresh and updated
- Use consistent formatting
- Add visual elements (images, diagrams)
- Write clear, concise descriptions
- Highlight achievements and impact
- Document your process
- Engage with your audience

## Support

For issues or questions about content management, refer to the Sanity documentation or contact support.
